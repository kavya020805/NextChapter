# Sprint Planning (Conflict-Minimized Flow)

## Sprint 1: Core Foundations
*Goal:* Make the platform usable (users can register, log in, and browse books).

- *Epic 1: Authentication* → US1–US5 (register/login/social login/password recovery/sign out)  
- *Epic 1: Profile basics* → US6–US7 (view + update profile)  
- *Epic 3: Catalogue basics* → US24–US25 (browse by genre, search)  
- *Epic 2: Admin basics* → US9–US10 (admin login + manage books)  

*Outcome:*  
Users can sign up, log in, browse/search books. Admins can populate catalogue.

---

## Sprint 2: Reading Essentials
*Goal:* Deliver the core book reading experience.

- *Epic 4:* US14 (open/read books), US15 (mark as read), US16 (wishlist/to-read)  
- *Epic 4:* US17 (view books read), US18 (cross-device sync)  
- *Epic 4:* US22–US23 (bookmarking, highlighting)  
- *Epic 4:* US55 (dark mode & font customization)  
- *Epic 2: Admin* → US11 (manage reviews) so users can safely interact  

*Outcome:*  
Readers can open, read, track progress, and personalize their reading.

---

## Sprint 3: Community & Engagement
*Goal:* Add social credibility and peer interaction.

- *Epic 5:* US19–US21 (rate, review, discuss)  
- *Epic 5:* US47 (like/upvote reviews)  
- *Epic 5:* US51 (report inappropriate content)  
- *Epic 2:* US46 (AI content moderation) + US48 (thread moderation tools)  

*Outcome:*  
Users can engage with community in a controlled, moderated way.
